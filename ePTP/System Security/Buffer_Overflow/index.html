  • DEP bypass
    • It is a hardware and software defensive measure for preventing the execution of code from pages of memory that are not explicitly marked as executable.
    • Bypassing DEP is possible by using a very smart technique called Return Oriented Programming ( ROP). ROP consists of finding multiple machine instructions in the program (called gadget), in order to create a chain of instructions that do something.
    • Gadget ‚Üí multiple machine instructions in the program.
      • Used to create a chain of instruction that do something.
      • Gadgets are small groups of instructions that perform some operations (arithmetical operations on registers, check for conditional jumps, store or load data and so on) and that end with a RET instruction.
    • The goal of ROP gadget is ‚Üí call a memory protection (Kernel API such as VirtualProtect), to mark the stack as executable; this will allow us to run our shell code,
      • But we can also use ROP gadgets to execute direct commands or copy data into executable regions and then jump to it.
    • In order to avoid the exploit of such techniques, ASLR was introduced. By making kernel API‚Äôs load at random addresses, bypassing DEP becomes hard.

  • Canary bypass
    • Canary‚Üí Location has a small random intger value.
    • The goal of Canaryis ‚Üí to modify almost all function‚Äôs prologue and epilogue to add the Canary before the return instruction (before epilogue), and check if there was a buffer overflow was happened.
      • The process is that the prologue adds a value before the epilogue, and the epilogue checks if this value was changed or not, if it was changed, that was a buffer overflow.
    • To bypass this technique you should guess or retrieve the Canary value and add it to the payload.
