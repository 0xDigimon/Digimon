4_Shellcoding

  • Execution of Shellcode is not via only EIP register, but can also execute via SEH (Structured Execution Handler).
    • Exception Handling) frame activates. The SEH frames store the address to jump to when there is an exception, such as division by zero.
  • Types of Shellcodes → denpeding on how shellcodes run and give control the attacker.
    • Local Shellcode
      • A Local shellcode is used to exploit local processes in order to get higher privileges on that machine.
      • known as Privilege escalation shellcodes
    • Remote Shellcode
      • Sent through the network along with an exploit.
      • known as RCE
      • The goal of this type is → provide remote access to the exploited machine.
      • Remote shellcodes can be sub divided based on how this connection is set up: • Connect back → initiates a connection back to the attacker’s machine • Bind shell → binds a shell to a certian port that the attacker can connect it. • Socket Reuse
    • Staged Shellcodes
      • shellcodes are used when the shellcode size is bigger than the space that an attacker can use for injection (within the process).
      • Staged shellcode may be localor remoteand can be sub divided into Egg hunt shellcode and Omelet shellcode.
        • Egg-hunt Shellcode
        • Omelet Shellcode

  • C language string functions work fine until it find a NULL-Byte (%00) it ends, so it will execute what is before the NULL-Byte.
  • Shellcodes Encoding
    • NULL-free encoding
      • Encoding a shellcode that contains zeros (%00), and replacing them with instructions that don’t have zeros and do the same task.
    • Alphanumeric and printable encoding
  • Creating Shellcode
    1. Find the target function address (with arwin.exe)
    1. Know what are arguments reuqired for this function.
    1. Write an ASM code that push arguments first, then call the function.
    1. Assmebling this code with nasm → nasm -f win32 sleep.asm -o sleep.obj
    1. Disassmebling the code to see the btyes of our instructions using objdump like this

      → objdump -d -Mintel sleep.obj

    1. Cleaning up the code by removing spaces and add /x prefix.

    • \x00 → is a string terminator → it causes a problem with NULL-free shellcodes.
    • To push a string to stack you should follow this:
      1. String should be just 4 bytes(split it if bigger than 4 bytes).
      1. Reverse the string (calc.exe → .exe, clac)
      1. convert the string to hexadecimal → (”\x2e\x65\x78\x65" → “.exe”)
      1. Add push btye in fornt of each line → \x68
      1. Last step is to terminate the string with \x00
      Note: when we pushing a byte the PUSH opcode will be \x6A, but when we pushing 
a word/dword the PUSH opcode will be \x68.
      Note: Function arguments need a pointer to the strings, so we save the strings
into registers.
- Every string will have a pointer using registers.
